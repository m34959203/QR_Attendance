# Политика безопасности

## Применимые версии

| Версия | Поддерживается |
|--------|---------------|
| 1.2.x  | ✅ Да          |
| 1.1.x  | ⚠️ Обновите    |
| 1.0.x  | ❌ Нет         |

## Реализованные меры безопасности

### Аутентификация
- Панель `/admin` и все `/api/*` маршруты защищены HTTP Basic Auth
- Пароль задаётся в `ADMIN_PASSWORD` через `.env`
- **Timing-safe сравнение** пароля через `crypto.timingSafeEqual` (защита от timing-атак)
- Rate limiting: 10 неверных попыток → блокировка IP на 15 минут
- **Обязательно использовать HTTPS** — Basic Auth передаёт credentials в Base64

### CSRF-защита
- Проверка `Origin`/`Referer` заголовка для всех POST/PUT/DELETE на `/api/*`
- Запросы с чужого Origin блокируются с HTTP 403

### HTTPS
- При `BASE_URL=https://...` автоматический редирект HTTP → HTTPS
- `X-Forwarded-Proto` проверяется для работы за reverse proxy

### Security Headers (Helmet.js)
- `Content-Security-Policy` — `default-src 'self'`, `frame-ancestors 'none'`
- `X-Frame-Options: DENY` — защита от clickjacking
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: no-referrer`

### Rate Limiting
- `/g/:groupId` (сканирование) — 10 запросов/мин с одного IP
- Авторизация — 10 неверных попыток → блок 15 мин
- API запись (POST/PUT/DELETE) — 60 запросов/мин
- Все лимиты: HTTP 429 с описанием ошибки

### SQL Injection
- Все запросы к SQLite через параметризованные выражения (`db.run(q, params)`)
- `ALTER TABLE` используется только с hardcoded значениями в миграциях

### XSS
- Все динамические данные экранируются через `esc()` (`& < > " '`)
- CSP запрещает inline-скрипты (кроме `unsafe-inline` для index.html)

### Токены и секреты
- Все токены хранятся в `.env` (в `.gitignore`)
- `config.js` не содержит секретов в коде
- Аудит-лог не записывает значения токенов
- `.dockerignore` исключает `.env` из Docker-образа

### Персональные данные (GDPR)
- Данные несовершеннолетних: требуется согласие родителей (`consentDate`)
- GDPR-удаление: полное стирание без возможности восстановления
- Срок хранения: автоочистка по `DATA_RETENTION_YEARS` (ежедневно в 03:00)
- Передача данных: только через выбранный WA провайдер и Telegram Bot API
- Родитель может отписаться: Telegram `/stop`, WhatsApp/Email — обращение к педагогу

### QR-коды и PIN
- ID учеников — UUID v4 (128 бит энтропии)
- PIN — 4 цифры, уникальный в пределах группы
- Защита от двойного скана: 10 минут блокировки
- Страницы сканирования: `noindex, nofollow`
- Кабинет родителя: токен = первые 12 hex UUID (48 бит)

### Docker
- Запуск от непривилегированного пользователя `node`
- `HEALTHCHECK` каждые 30 сек через `/health`
- `.dockerignore` исключает `.env`, `node_modules`, `data/`, `.git`

### Процесс
- Graceful shutdown: SIGTERM/SIGINT → остановка таймеров → закрытие сервера
- Логирование ошибок в `data/error.log` с timestamp, URL, IP

## Сообщить об уязвимости

Если вы обнаружили уязвимость:
1. **Не создавайте публичный Issue**
2. Опишите уязвимость владельцу системы напрямую
3. Укажите: описание, шаги воспроизведения, возможные последствия
4. Ожидайте ответа в течение 72 часов
