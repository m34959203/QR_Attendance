# Руководство по разработке

## Философия проекта

Система специально остаётся простой: один Node.js процесс, один файл фронтенда, минимум зависимостей. Не усложняй без необходимости.

## Правила разработки

### Обязательные

1. **Вся работа с БД — только через `db.js`**. Нельзя использовать sql.js напрямую в других модулях.
2. **WhatsApp-сообщения — только через `wa.send()` или `wa.sendDirect()`**. Прямые вызовы green-api запрещены.
3. **`public/index.html` — один файл**. Не делить на отдельные CSS/JS файлы.
4. **Формат телефона — только цифры: `77001234567`** (без +, пробелов, скобок).
5. **Все времена хранятся в UTC, конвертируются при отображении** через `config.TIMEZONE`.
6. **`config.js` — единственный источник настроек**. Не читать `process.env` напрямую.
7. **`db.init()` — async**. Вызывается один раз в IIFE в `server.js` до старта сервера.

### Рекомендуемые

- Без новых npm-зависимостей если можно обойтись встроенными Node.js модулями
- Все новые API-маршруты — под `/api/`, защищены `authGuard`
- Публичные маршруты (как `/scan`) — явно задокументированы
- Новые действия администратора — фиксировать через `db.audit()`

## Структура коммита

```
тип: краткое описание на русском

Подробное описание если нужно (необязательно).
```

**Типы:**
- `feat:` — новая функциональность
- `fix:` — исправление ошибки
- `docs:` — только документация
- `refactor:` — рефакторинг без изменения поведения
- `security:` — исправление безопасности
- `chore:` — обновление зависимостей, конфиги

**Примеры:**
```
feat: добавить фильтр посещаемости по дате
fix: исправить двойной скан при Telegram webhook
docs: обновить CHANGELOG для версии 1.1.0
security: добавить rate limiting на /scan/:id
```

## Запуск в режиме разработки

```bash
cp .env.example .env
# Заполнить BASE_URL=http://localhost:3000
npm run dev   # node --watch src/server.js
```

## Тестирование изменений

Перед коммитом проверить вручную:
- [ ] Добавить ученика — QR генерируется
- [ ] Скан по адресу `/scan/:id` — запись создаётся
- [ ] WhatsApp тест — сообщение отправляется (если настроен)
- [ ] Telegram тест — сообщение отправляется (если настроен)
- [ ] Экспорт Excel — файл скачивается
- [ ] Аудит-лог — действия фиксируются

## Добавление новых маршрутов

```javascript
// ── Шаблон нового маршрута ──
app.get('/api/some-resource', (req, res) => {
  // req.adminIp доступен после authGuard
  const data = db.someMethod();
  db.audit('action', 'entity', '', 'details', req.adminIp);
  res.json(data);
});
```

## Добавление новых столбцов в БД

Добавить в `_migrate()` в `db.js`:
```javascript
try {
  db.run('ALTER TABLE students ADD COLUMN newField TEXT DEFAULT ""');
  _save();
} catch {} // Игнорируем если уже существует
```

## Обновление документации

При добавлении нового функционала обновить:
1. `CHANGELOG.md` — добавить в раздел Unreleased
2. `CLAUDE.md` — обновить схему БД или API если нужно
3. Соответствующий .docx документ при следующем релизе
