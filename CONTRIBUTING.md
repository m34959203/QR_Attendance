# Руководство по разработке

## Философия проекта

Система для частных кружков и учебных центров. Специально остаётся простой: один Node.js процесс, один файл фронтенда, минимум зависимостей. Не усложняй без необходимости.

## Правила разработки

### Обязательные

1. **Вся работа с БД — только через `db.js`**. Нельзя использовать sql.js напрямую в других модулях.
2. **WhatsApp-сообщения — только через `wa.send()` или `wa.sendDirect()`**. Прямые вызовы провайдеров запрещены.
3. **`public/index.html` — один файл**. Не делить на отдельные CSS/JS файлы.
4. **Формат телефона — только цифры: `77001234567`** (без +, пробелов, скобок).
5. **Все времена хранятся в UTC, конвертируются при отображении** через `config.TIMEZONE`.
6. **`config.js` — единственный источник настроек**. Не читать `process.env` напрямую.
7. **`db.init()` — async**. Вызывается один раз в IIFE в `server.js` до старта сервера.
8. **Пароли** — сравнивать только через `crypto.timingSafeEqual`.
9. **HTML** — все динамические данные через `esc()`, без исключений.
10. **Терминология** — занятие (не урок), педагог (не учитель), группа (не класс).

### Рекомендуемые

- Без новых npm-зависимостей если можно обойтись встроенными Node.js модулями
- Все новые API-маршруты — под `/api/`, защищены `authGuard`
- Публичные маршруты — явно задокументированы в CLAUDE.md
- Новые действия администратора — фиксировать через `db.audit()`
- Новые столбцы в БД — добавлять через `_col()` в `_migrate()`

## Структура коммита

```
тип: краткое описание

Подробное описание если нужно (необязательно).
```

**Типы:**
- `feat:` — новая функциональность
- `fix:` — исправление ошибки
- `docs:` — только документация
- `refactor:` — рефакторинг без изменения поведения
- `security:` — исправление безопасности
- `chore:` — обновление зависимостей, конфиги

**Примеры:**
```
feat: add lesson reminders 1h before class
fix: prevent double-scan race condition
docs: update API documentation for v1.2.0
security: add CSRF origin check for API mutations
```

## Добавление новых маршрутов

```javascript
// ── Шаблон нового маршрута ──
app.post('/api/some-resource', (req, res) => {
  // req.adminIp доступен после authGuard
  // Валидация входных данных:
  if (!req.body.field) return res.status(400).json({ error: 'Описание ошибки' });

  const data = db.someMethod();
  db.audit('action', 'entity', data.id, 'details', req.adminIp);
  res.json(data);
});
```

## Добавление новых столбцов в БД

Добавить в `_migrate()` в `db.js`:
```javascript
_col('students', 'newField', 'TEXT DEFAULT ""');
```

Функция `_col()` использует `ALTER TABLE ... ADD COLUMN` и игнорирует ошибку если столбец уже существует.

## Добавление нового WhatsApp провайдера

1. Создайте файл в `src/whatsapp-providers/new-provider.js`
2. Реализуйте метод `sendMessage(phone, text)` → `{ idMessage }`
3. Добавьте кейс в `whatsapp.js → init() → switch`
4. Добавьте переменные в `config.js` и `.env.example`

## Обновление документации

При добавлении нового функционала обновить:
1. `CHANGELOG.md` — добавить в раздел текущей версии
2. `CLAUDE.md` — обновить схему БД, API маршруты, правила
3. `docs/API.md` — если добавлены/изменены эндпоинты
4. `docs/DATABASE.md` — если изменена схема
5. `README.md` — если добавлена крупная фича
